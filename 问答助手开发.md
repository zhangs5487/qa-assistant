> **最后更新**: 2026-06-29

## 项目目标

构建面向外部用户的智能问答系统，数据来源于 `https://cqaip.cn/`（重庆市人工智能公共服务平台 + 建邺OPC社区）。

### 数据源 (cqaip.cn 11个API端点)

| 来源 | 端点 | 数量 | 认证 |
|------|------|------|------|
| 政策文档 | /api/policies | 16 | 无 |
| 行业资讯 | /api/industry-news | 34 | 无 |
| 赛事活动 | /api/competitions | 12 | 无 |
| 数据集 | /api/datasets | 2,176 | Casdoor |
| 供需市场 | /api/marketplace | 158 | Casdoor |
| 模型 | /api/models | 371 | 无 |
| + 社区/应用/配置等 | 其他 | ~8 | 混合 |

### 管线流程

```
REST APIs → api_fetcher.py (分页+重试+认证)
    → data/raw/*.json
    → parser.py (按来源解析)
    → CleanDocument → data/clean/documents/
    → chunker.py (LangChain, 500tk/50ov, 中文分句)
    → Chunk → data/clean/chunks/
    → qa_augmenter.py (GLM-4改写, 每QA→3变体)
    → qa_dedup.py (Embedding去重, threshold=0.92)
    → 438 QA对 → data/clean/qa_deduped/
    → build_knowledge_base.py / rebuild_all.py (BGE-M3 embed + Milvus)
    → Milvus: qa_pairs (438行) + doc_chunks (3,517行)
    → HybridRetriever: 向量检索(BGE-M3) + BM25关键词检索(jieba) + RRF融合(k=60)
    → Cross-encoder Re-rank (BGE-Reranker-v2-m3, 仅QA路径)
```

## 关键技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 爬虫 | REST API 直调 (非Scrapy) | 逆向工程后发现所有数据都是JSON API |
| 向量库 | Milvus (pymilvus 3.0 + milvus-lite 3.0) | 领导指定，生产级向量库 |
| QA增强 | GLM-4改写 ×3变体 | 覆盖口语/书面/简短/详细四种问法 |
| 导航知识 | 从site-config直接生成QA | 权威、精确、不依赖LLM幻觉 |
| 去重策略 | Cosine similarity >= 0.92 | 保留语义变体，去除完全同义 |
| 分块策略 | 500 tokens / 50 overlap | 中文优先按句号/换行分割 |
| **检索策略** | **向量(BGE-M3) + BM25(jieba) + RRF(k=60) + Cross-encoder Re-rank** | **混合召回 + 语义重排序** |
| **Re-rank策略** | **QA路径：BGE-Reranker (8候选)；RAG路径：跳过(LLM自己就是rerank)** | **QA需要精确重排，RAG用LLM兜底更适合** |
| **本地模型** | **全本地化：BGE-M3 + BGE-Reranker + Qwen3-1.7B** | **零API成本，数据不出本机** |
| **意图分析** | **关键词 + Embedding双重判定** | **先分类后路由，避免导航Q匹配内容A** |
| **来源链接** | **数据源URL + 政策详情页ID拼接** | **让用户能直接查看原文PDF/页面** |

### 如果改用标准 Milvus（非 Lite）

修改 `libs/storage/src/storage/milvus_store.py`：

```python
# 当前（Milvus Lite）：
MILVUS_DB_PATH=./data/milvus.db

# 改为（标准Milvus服务）：
MILVUS_DB_PATH=http://milvus:19530
# 同时注释 config/milvus_aliases.py 中的 lite 配置
```

`milvus_store.py` 中 `MilvusStore.__init__` 会自动识别 URI 格式：本地路径 → Lite，`http://`→ 标准Milvus。无需改代码。

### 无需修改的文件

```
config/settings.py        # 配置项从 .env 读取，无硬编码
libs/llm/providers/       # provider 由 factory 按配置选择

```
